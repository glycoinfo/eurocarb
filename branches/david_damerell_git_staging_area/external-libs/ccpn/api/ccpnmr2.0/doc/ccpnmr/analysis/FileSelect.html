Memops:FileSelect

  
Popups: Memops

FileSelect
GeneralEditor

The FileSelect is used in several places within Analysis to select a file or files for loading/saving.

The main table lists the contents of the current directory. File or files for loading (usually only a single file however) are selected by clicking on their name in this table. Double clicking on a directory in the table causes the current director to be set to the that directory. The table is then updated accordingly.

File name: <text_entry>

Here the user can enter the name of the file. If the name of an existing file is selected in the main table, this entry box is automatically filled in with the selected name.

Directory: <text_entry>

The path to the currently listed directory. The user may enter the name of a director here (pressing <Return> updates the main table) or the name may be set automatically according to the directory chosen in the main table or director buttons below.

Files of type: <list>

Allows the user to restrict the displayed file names to those with a specified suffix/wildcard match. The default is to display all files "All (*)".

Directory path: <list>

Allows the user to set the current directory by choosing from the hierarchy of directories that navigate the file system to the current location.

[ Up ]

Sets the directory to the one containing the current location (up the hierarchy).

[ New ]

Makes a new sub-directory within the current location. Prompts the user to enter the name of the new directory.

[ Refresh ]

Update the main table and file entries according to the current contents of the location.

[ Open/<commitMessage> ]

This option commits the selected file name(s) and directory to whichever function requested, e.g. to then open the file.
