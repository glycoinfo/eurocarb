This popup displays NMR information for all of the experiments within the current project.

| # |

The serial number of the experiment in the project.

| Name |

The name of an experiment. This column is editable by double clicking.

| Shift List |

Double clicking this column sets the shift list which will be used for an experiment. This is the list of chemical shifts which will be used for the resonances assigned to peaks in an experiment. A new shift list may also be created here.

Different experiments may have different shift lists if they were performed under different conditions. A given resonance may be represented in several experiments with different shift lists. In this instance the resonance has many chemical shifts, one for each shift list and one for each hence set of conditions.

| Data Sources |

The names of the data sources (e.g. spectra) which were recorded as a result of an experiment.

| Mol Systems |

The molecular systems to which the peaks of an experiment may be assigned. When an experiment is associated with a molecular system, if a peak within the experiment is subsequently assigned to a resonance with atoms in a different system the user will be warned about the action.

| Details |

A user specified data field. This column may be edited by double clicking.

[ Experiment Details ]

This opens EditExperimentDetailsPopup for the selected experiment.

[ Measurement Lists ]

This option opens a popup to display the measurement lists which are currently valid for (and may have been generated using) the currently selected experiment. Opens EditMeasurementListsPopup.

[ Make New ]

Makes a new arbitrary experiment object within the current project. This option prompts the user to input the number of data dimension that the new experiment will have. Such a new experiment object may subsequently be edited to specify the details of the experimental procedure and may have NMR spectra associated with it.

[ Delete ]

Deletes the currently selected experiment and any spectra, peaks etc. which derive from it. This does not affect in any way the files which store spectral data.
