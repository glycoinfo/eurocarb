There are too many experiments for any systematic naming scheme to cover all possibilities. These rules give a systematic name for many of them, and we recommend sticking to them whenever possible. There will be cases where a free-form text name is better (or unavoidable). Even though we do not cover all possibilites the system is still quite complicated. Hopefully it will be easier to interpret existing names than to make new ones.

- 1 There is only one ExpPrototype for every magnetisation transfer path, even where this would result in a 5D or 6D experiment. The various 2D, 3D, 4D, projections, reversed direction, reduced dimensionality or coupled/decoupled variants etc. are RefExperiments under the same ExpPrototype. E.g. the 5D HCACONH ExpPrototype would cover 4D HCAcoNH, 3D HncoCAH, etc.

- 2 Experiments are normally named by concatenating the AtomSiteTypes that describe the magnetisation transfer path ( e.g. HCACONH). You are free to use atom names that do not correspond to existing AtomSiteTypes where this is easier to understand, e.g. HNCAHA instead of HNCAH or HBCBCGCDHD, instead of HCaliCaroCaroH. Note that HN means 'H then N', and *not* 'peptide backbone proton'.

- 3 By default transfer is by J coupling or one-bond transfer. Other transfer types are indicated by putting an underline in the transfer path, and appending the transfer type to the end of the name. E.g. H_H.TOCSY, H_H_H.TOCSY,NOESY, HC_CH.TOCSY. TROSY transfer is not distinguished from other kinsd of one-bond transfer.

- 4 Out-and-back segments are indicated by putting the out-and-back trace in square brackets. E.g. H[N]_H.NOESY is the normal HSQC-NOESY, H[C]_H[C].NOESY the 4D carbon-carbon NOESY. In HNCAHA magnetisation starts on backbone H and ends on HA, whereas the out-and-back version is called H[N[CA[HA]]].

- 5 Some experiments may have several possible nuclei for a given axis, or several alternative magnetisation transfer paths. These are shown by putting the alternatives in curly braces. If the possibilities are mutually exclusive, they are separated by a vertical line. Examples would be e.g. H[{N|C}]_H.NOESY, an HSQC-NOESY with both C and N in parallel on the second axis, or H{CA|Cca}CONH, the protein backbone experiment more commonly known as HBCB/HACACONNH. Separate transfer paths that both followed in parallel are also put in curly braces, but separated by a plus sign. Examples would be H{[N]+[HA]}, the HNHA experiment, or H_H{[c(0)]+[n(0)]}.NOESY, a NOESY with protons bound to unlabelled heavy atoms selected in the second dimension.

- 6 AtomSites that are part of the coupling network but are not actually part of any ExpMeasurement are given in lower case. This is used for experiments that filter on coupling partners without necessarily transferring magnetisation to them, such as the H[C[co]] CO-filtered HSQC. It is also used for atoms in the transfer path that cannot be sensibly measured, as in the H{CA|Cca}CONH (HBCB/HACACONNH) experiment. The latter usage is the same as used for lower-dimensionality RefExperiments (see below).

- 7 If you want to indicate the number of atoms of a given type that serve as coupling partners you simply affix the number, in parentheses. A 3D HSQC-NOESY set up to suppress signals starting on CH2 groups would be HC[h(2n+1)]_H.NOESY. An HCC-3D set up to select quaternary carbons in the third dimension would be H[C[C[h(0)]]]

The most complex examples you can imagine are probably the double-half filtered NOESYs - fortunately there are not that many of them. A double-half-filtered 15N/13C NOESY, which serves to select intermolecular NOEs between doubly labeled molecules and unlabeled molecules is be written { H[{n|c}]_H{[n(0)]+[c(0)]} | H{[n(0)]+[c(0)]}_H[{n|c}] }.NOESY.

- 8 Non-shift dimensions are given as their type followed by the AtomSites in parentheses. E.g. J(HH) (proton-proton coupling), DQ(CN) (carbon-nitrogen double quantum magnetisation).

- 9 RefExperiments are specific variants of their ExpPrototypes. While it is not possible to cover all possibilities it is recommended to follow these rules. Extra information can always be added as a dot-separated suffix.:

Lower-dimensional subsets: The most common RefExperiments are simply cases where not all possible dimensiopns are measured. These are named by writing in lower case the dimensions that are not measured. E.g. the 5D HCACONH ExpPrototype would give 4D HCAcoNH, 3D HncoCAH, etc

Couplings: Couplings will often be unresolved, and will rarely be of interest in the analysis. The 'coupled isotopes' attribute is only relevant if there are couplings that need to be measured and assigned. The recommended naming style is e.g. 'HC.C coupled to CO', or 'HNCAH.H1,H2 coupled to C'.

Reversed experiments: The ExpPrototype describes the magnetisation transfer path. so that e.g. HNCAH and HCANH will be described in the same ExpPrototype. The recommended naming style is simply to reverse the name (as shown here) and set isReversed to True, while leaving the rest of the RefExperiment description unchanged.

NOESY and ROESY: These are indistinguishable as far as transfer paths etc. are concerned. For convenience ROESY is a RefExperiment under the H_H.NOESY ExpPrototype.

Reduced-dimensionality and projection experiments: Here several measurements from the ExpPrototype are measured in the same dimension as a linear combination. Here the total dimension of the experiment is added as a suffix, followed by a curly brace describing the composition of the combined axis. Examples: HNCOCA, CA as splitting on CO axis: H[N[CO[CA]]].3D.{CO:1;CA:0.5,-0.5} HNCOCA, N-CO+2CA projection: H[N[CO[CA]]].2D.{N:1;CO:-1;CA:2} HNCOCA, N+CO projection: H[N[CO[ca]]].2D.{N:1;CO:1} In the unlikely case that more than one dimension is acquired as a combination, it is possible to add more than one curly brace suffix.
Examples of this kind of name can be found in the NmrExpPrototype distribution for the H[N[CO[CA]]] and the H{CA|Cca}CONH experiments.
Note that projection experiments may be treated as individual DataSources under a single Experiment. In that case the NmrExpPrototype given to the Experiment is that of the full-dimensionality version, and the DatSources do not have individual RefExperiments. The appropriate scaling factors are stoerd in the Nmr.DimensionScaling class.

TROESY: TROESY is strictly speaking a combination of a shift and a coupling and could be described as such. However this would require that TROESY and non-TROESY versions were described by separate NmrExpPrototypes, and would give horribly complex names. Instead TROSEY is considered as a subtype of onebond and is not described in the experiment prototype description at all. If desired it can be added as a '.TROESY' suffix. The precise type of transfer can be specified in ccp.nmr.Nmr.ExpTransfer.transferSubType.
