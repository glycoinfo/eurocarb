CcpNmr applications like Analysis and FormatConverter use the Python programming language API (application programming interface). This means that the data for these programs and any scientific project is stored as XML files. Each working project will consist of several XML files, each of which correspond to a different Data Model package. This means for example that NMR assignment data is stored separately from Structures which are again separate from Molecule descriptions etc. The advantage of this system is that a CcpNmr application needs to only load and save the packages of data that it requires at the time, thus decreasing load/save times and memory requirements. A consequence of this is that the data ends up in different locations on disk. You may need to be aware of this, if for example you copy a project from one location to another. However, a program like Analysis will prompt you if it cannot find an XML file and you can then redirect the project to the new location.

File Locations

There is one 'master' project file which is stored locally and knows the location of all other package files. The organisation of the other XML files, although totally configurable, by default corresponds to:
  • Editable information (assignments, sequence etc.) being stored within a directory with the same name as the project. This directory lives next to the project XML file itself. Within the project directory there are further sub-directories which correspond to the Data Model classification of the packages.

  • Immutable, reference information (chemical components, chemical element data, etc.), stored in a library corresponding to part to the CCPN installation directories, e.g. $CCPN_HOME/data/.
