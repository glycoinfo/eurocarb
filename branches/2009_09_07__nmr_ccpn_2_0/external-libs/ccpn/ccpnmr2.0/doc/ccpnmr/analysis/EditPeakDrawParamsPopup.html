This popup allows the user to specify how the peak cross symbols are drawn and annotated within Analysis.

- Peak size -

This table is used to set how the drawn size of a picked peak symbol (usually a cross) is determined.

<list>

This pulldown allows the user to select whether the crosspeak size is a set number of screen pixels, a set ppm range, scaled globally according to intensity (relative to all spectra) or scaled relative to the intensities in a peak list.

- Peak annotation -

These options determine how a peak is annotated, according to any assignments, within Analysis.

Annotations of existing peaks are not updated automatically (excepting the Minimal Annotations option) because altering peak annotations is a potentially time-consuming process. The user must explicitly activate the refreshing procedure with the [ Update Full Annotations ] button.

[] Spin System Info

If set to ON peaks will carry an indication of which spin system any connected resonances belong to. For example a peak annotated "16GlyHa2 {12}[30]" has two spin system indicators; "16Gly" for the first dimension and "{12}" for the second dimension. Here "{12}" simply says that the spin system is number twelve, i.e. it is not known which residue it pertains to.

[] Atom Assignment

If set to ON, peaks will carry details of the atoms to which any connected resonances may be assigned to. For example a peak assigned to the alpha and amide resonances of 16Gly may carry the annotation "16GlyH 16GlyHa 16GlyN" thus stating the atoms to which the peaks is connected. If OFF the peak will be annotated according to (arbitrary) resonance numbers only.

[] Chain Assignment

If set to ON, peak annotations will carry letters indicating the molecular chain that its resonances are assigned to. For example a NOE peak between 16Gly amide proton of chain A and 77Ser amide proton of chain B may carry "A16GlyH B77SerH".

[] Molecular System

If set to ON, this means that a peak's annotation will be prepended with the code of any molecular system to which it may be assigned, e.g. "MS1: 16GlyH 16GLyN", for a molecular system with code "MS1:".

[] Details

If set to ON, this means that a peak's annotation will carry any user entered details. For example, "MS1: dodgy peak A16GlyH B77SerH", where "dodgy peak" is the user specified value.

[] Minimal Annotations (overriding option)

If set to ON, this overrides all other options. Instead of displaying atom, spin system, chain information etc. the annotation is changed (reversibly) to a simple string to describe which residue (using one letter codes) the peak is assigned. For example "MS1: 16GlyH 16GLyN" will become "16G".

[ Update Full Annotations ]

Update all of the peaks within the current project according to the rules stated above. Otherwise the rules will only apply to new peaks. Note that this option may take a long time if the current project carries many peaks.

- Peak merit symbols -

These options determine how a symbol (or characters) are added to a peak's annotation according to its merit value.

Good merit (>0.66) <text_entry>

Sets the symbol for peaks with a merit value over 0.66: These peaks are those that are generally deemed worthy of consideration.

Medium merit <text_entry>

Sets the symbol for peaks with intermediate merit values: These peaks are those that are generally deemed to be of dubious quality or worth.

Poor merit (<0.34) <text_entry>

Sets the symbol for peaks with low merit values (less than0.34): These peaks are those that are generally deemed incorrect or unreliable for a particular function (e.g. constraint generation).

[ Set Symbols ]

This option sets the merit symbols at the currently entered values. This will affect any new peak or existing peaks if their annotation is updated.
