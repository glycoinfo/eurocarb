Allows the user to specify the sequence of a biological polymer molecule. Using this sequence a molecule template is created. From this template assignable chains may be made using EditMolSystemsPopup.

- Options -

Molecule Type: <list>

Selects which class of polymer sequence is being entered for. Currently one of protein, DNA or RNA. Changing the molecule type to and from protein will clear any entered sequence. If the type changed between DNA and RNA the entered sequence will be preserved, but thymidine residue will be swapped for uracil ones.

Numbering starts at: <text_entry>

Allows the user to set the number for the start of the entered sequence of polymer residues.

Ccp Codes: <list>

Lists all of the Ccp Codes (often the same as normal residue three letter codes) for residues which can be added for the selected molecule type. This pulldown list also sets witch residue will be appended to the sequence by using the adjacent button.

[ Append Residue ]

Add a sequence element to the end of the input sequence. The type of residue code added corresponds to the option selected in the adjacent pulldown menu, described above.

<> Three letter codes

When ON states that the entered sequence is in three letter code form, delimited by white space. If the sequence was previously one letter codes it will be re-rendered in this form. The actual codes used are the Ccp Codes which may be longer than three letters for unusual residues. As long as such codes are bounded by white space they will be interpreted.

<> One Letter Codes

When ON states that the entered sequence is described by residue single-letter codes. This may change an existing sequence to this form. If a residue is not a common one for the molecule type and had no single-letter code the represented the a Ccp Code in square brackets. For example phosphotyrosine is represented, or may be entered, as [PTR].

- Sequence input -

This text entry window allows the user to type and paste in the desired polymer sequence, either as one or three letter codes as set above.

[ OK ]

Commits the entered sequence (passes the sequence to whichever function requested input) and closes the popup.

[ Tidy ]

Renders the entered sequence in a neat format with numbered sequence lines.

[ Read File ]

Opens a file browser so the user can select a file to read a sequence from. The contents of such a file are placed within the sequence window, the same as if the sequence had been typed or pasted. The loaded sequence may then be edited before commitment.

[ Clear ]

Clears the text window of any input sequence.
