Assignment steps

Assignment in CcpNmr Analysis is a two-step process. A traditional view of NMR assignment is as the connection of chemical shifts to particular atoms. This is still true of the CCPN Data Model, but by recording several intermediate steps between chemical shift and atom it is able to represent ambiguous, intermediate and otherwise poorly-defined assignment status. The critical point within these intermediate steps is a central CCPN Data Model object called Resonance.

Resonances

The Data Model Resonance object has no obvious real-world counterpart (unlike Atom, Spectrum etc.). It can be thought of as both a stand-in for a an atom or group of atoms (i.e. even anonymous) or a tag for chemical shifts. However, it is more proper to think of it as the phenomenon arising from the resonance of atom(s) which gives rise to the experimentally recorded chemical shift(s). It does not represent a single chemical shift because the value of chemical shift varies with according to experimental conditions. It does not represent a specific atom because multiple equivalent atoms can give rise to a single signal. Also, the resonating atoms may not be known, and yet the phenomenon can still be identified and correlated within and across spectra.

Accordingly, within Analysis the two steps of assignment are:

1 The connection of chemical shifts, represented by crosspeak positions in a given dimension, to a Resonance object. Connection of a peak dimension to multiple Resonances results in ambiguous assignment.
2 The connection of atoms in atom sets to Resonances. Atom sets represent NMR equivalent groups like methyls and fast rotating aromatic rings. When atom sets are connected to more than one Resonance the situation represents the ambiguity involved with prochiral atoms; for example an HB2 could have either one of two hydrogen beta shifts, but its not known which.
