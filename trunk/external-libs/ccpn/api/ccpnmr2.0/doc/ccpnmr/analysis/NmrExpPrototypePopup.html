This popup allows the user to create and edit experiment prototypes. Experiment prototypes are reference descriptions of pulse programs, used to initialise an Nmr.Experiment. The description includes the atoms involved in the pulse sequence, the transfers (coupling, NOESY, ...) between them, the measurements recorded (shifts, couplings etc.), and the order of traversal. The ExpPrototypes are always the highest possible dimensionality you can get from the transfer path, even if that makes for a 5D or 6D experiment. Different RefExperiments describe different experiment variants, such as lower dimension projections.

ExpPrototypes and RefExperiments have semi-systematic names. A detailed description of the naming system may be found in the NmrExpPrototype nomenclature page.
- The simplest case is a straight-through experiment with only J-coupling transfer, where you simply list the atoms, e.g. (5D) HCACONH.
- Atoms on the transfer path that are not measured are in lower case, e.g. (3D) HCAconH.
- Different transfers are shown as in H_H_H.TOCSY,NOESY, a 3D H-H-H tocsy-noesy experiment
- Atoms in out-and-back transfers are put in square brackets, e.g. H[N]_H.NOESY (a 3D HSQC-NOESY).
- Where there are several alternative on the path, they are put in curly braces. Mutually exclusive paths are separated by a vertical bar, e.g. H[{N|C}]_H.NOESY a 3D HSQC-NOESY where the heavy atom may be either C or N). Parallel paths are sepatated by a comma, e.g. H[{N,HA}] (the HNHA experiment).

- NmrExpPrototypes -

This table displays the various experiment prototypes present within the current project. The column names should be self-explanatory

| Name |

Experiment name in (semi) systematic nomenclature.

| maxDim |

Maximum number of dimensions in the RefExperiments belonging to the ExpPrototype. By convention there will be a RefExperiment with the maximum possible number of dimensions, even if this makes for a 5D or 6D experiment.

| Synonym |

Commonly used or explanatory name, non-systematic.

| Keywords |

Keywords about the experiment or teh context it is used in (e.g. 'DNA').

| Details |

Detailed explanation of the experiment; extra information.

| isEditable |

This column says if the experiment may be modified, or if it is frozen.

[ Create New ]

Adds a new ExpPrototype with default parameters. The parameters can be changed by editing them afterwards.

[ Create Copy ]

Adds an ExpPrototype that is a copy of the currently selected one. The parameters can be changed by editing them afterwards.

[ Freeze ]

Freezes the experiment prototype so it can no longer be modified, except by adding new RefExperiments. Experiment prototypes are reference information. They should generally not be changed once made, as there may be experiments using them. Experiment prototypes that are loaded from file are in a frozen state. The ony way to unfreeze a prototype is via the command line. To unfreeze the prototype with serial 5 (e.g.) you do
myProject = top.project
myPrototype = myProject.findFirstNmrExpPrototype(serial=5)
myPrototype.isEditable = True
myPrototype.isModifiable = True
after which you click some editable field to update the popup.
This procedure should only be used if you know what you are doing - that is why it has been made so deliberately obscure.

[ Delete ]

Deletes the currently selected experiment prototype.

- AtomSites -

The table shows the atom sites that are involved in magnetisation transfer. This is not the same as the measurements - there may be nuclei used only for filtering, or you may be measuring other parameters than shifts.

| Isotope |

The Isotope of the AtomSite.

| Name |

The name of the AtomSite should reflect the isotope or the type. The protototype and refExperiment names are generally made from the AtomSite names.

| min Shift |

| max Shift |

The minimum or maximum shift (in ppm) that the nucleus may have. Should be set only where filtering or frequency-specific pulses are used to limit the frequency. Serves e.g. to distinguish HNCA from HNCO. It is recommended to use the values from the pre-existing types whenever possible.

| min Number |

The minimum number of nuclei that may be present. Used for experiments that select nuclei with a minimum number of coupling partners, e.g. non-quaternary carbons.

| max Number |

The maximum number of nuclei that may be present. Used for experiments that select nuclei with a maximum number of coupling partners, e.g. CH2 carbons.

| number Step |

Used for experiments that select e.g. carbons bound to an odd number of protons. This specific case would have minNumber=1, numberStep=2.

[ Create New - Type: ]

Add a new AtomSite of the selected Type. The type provides default settings for the parameters. The parameters can be changed by editing them afterwards.

[ Delete ]

Deletes the currently selected AtomSite.

- ExpMeasurements -

The table shows the values that may actually be measured in the experiment, or that are used as stepping stones in the transfer path, whether measured or not. Typically you will be measuring the shift of a single AtomSite, but you could measure other parameters that might involve several AtomSites, such as coupling constants or multiple quantum coherences.

| Type |

The kind of parameter you are measuring.

| atomSites |

The atomSite or sites your parameter belongs to. How many you can have depends on the measurement Type

| atomSiteWeights |

This is only relevant for MQ shifts. Serves to distinguish between e.g. three-spin triple quantum magnetisation (atomSites H1,H2,H3, weights 1,1,1) and three-spin single quantum magnetisation (atomSites H1,H2,H3, weights 1,1,-1). If atomSiteWeights is set, the number of weights must be the same as the number of atomSites

[ Create New ]

Add a new ExpMeasurement, corresponding to the shift of the currently selected AtomSite. The parameters can be changed by editing them afterwards.

[ Delete ]

Deletes the currently selected ExpMeasurement.


[ ExpGraphs ]

Open the ExpGraph editor popup, for the current NmrExpPrototype. This lets you edit the magnetisation transfers between AtomSites and the order in which they are traversed.

[ RefExperiments ]

Open the RefExperiment editor popup, for the current NmrExpPrototype. This lets you create and edit the individual experiment variants that correspond to the ExpPrototype.
